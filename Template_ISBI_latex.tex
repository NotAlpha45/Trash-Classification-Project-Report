\begin{abstract}

With the increasing use of multimodal data and specialized models for classification tasks, important questions arise regarding the effectiveness of traditional inference-based models versus retrieval-augmented approaches, especially when it comes to data skewness and domain drift. In this work, we compare supervised multimodal models with retrieval-augmented classification using 3 novel vector-based heuristics for a four-class garbage classification problem. Through our various experiments, we found that, while supervised fusion models provide strong baseline performance and inference velocity, retrieval-based methods, especially density-aware approaches, show improved robustness and better performance (upto 3.41\% in F1 Score, without any additional tuning).

\end{abstract}

\section{Introduction}
\label{sec:intro}

Garbage classification is an intriguing field that offers a good opportunity for applied multimodal classification tasks, as waste management facilities regularly use machine learning \cite{alexakis2025enhanced} to sort garbage according to some predefined criteria. Deep learning (DL) models have been a popular choice in multimodal classification due to their ability to learn from heterogeneous data sources such as images and text and being more adaptable for complex patterns in both unimodal and multimodal scenarios. By leveraging complementary information across modalities, multimodal approaches often outperform unimodal systems \cite{alexakis2025enhanced}. However, these supervised approaches may face limitations under challenging conditions such as class imbalance, limited training data or domain drift. Those conditions often require rigorous training, normalization, complex feature engineering, augmentation, or re-training an existing model - which means a penalty of computational resources on top of an already extensive dataset required for these models to work properly \cite{islam2026learning}.

An alternative approach is retrieval-augmented classification (RAC), which utilizes embedding-based memory to retrieve similar instance and make predictions based on neighborhood evidence. This allows the model to incorporate additional contextual information beyond the training distribution but introduces challenges in retrieval design and scoring. Despite increasing interest in such methods, a systematic comparison between traditional supervised inference and retrieval-based approaches remains limited. In our work, a two-phase framework is presented to evaluate both strategies for multimodal garbage classification, with experiments conducted under standard settings, class imbalance, and varying memory size to analyze performance, robustness, and generalization. Implemented with modern vector databases like Chroma, Pinecone, LanceDB, and PGVector - RAC only has a storage overhead, which is much cheaper than memory in terms of hardware price. 

To address the challenges in retrieval design and scoring, we propose a novel heuristic - Density Normalized Distance Score (DNDS) that specifically addresses the challenge of data skewness or imbalance, and domain drift by taking into account class distributions in various ways. This DNDS has 3 variants (which ultimately makes it 3 heuristics) that we studied in this work - Global DNDS, Local DNDS, and KDE DNDS (The details of which are discussed in \autoref{sec:methodology}). This DNDS heuristic is dynamic (works well even with continuously increasing database size), designed to work with cosine distances with high-dimensional vector embedding, and corrects itself for class skewness. Throughout this study, we focused more over evaluating these 3 variants of the DNDS heuristic, with a traditional deep learning approach involving a 2 step approach implemented with MobileNetV3 and DistilBERT. Now, our research questions (RQ), that we tried to answer in this study is as follows:  

\begin{itemize}
  \item \textbf{RQ 1}: Does DNDS have better performance compared to traditional deep learning methods for classification? 
  \item \textbf{RQ 2}: How robust is DNDS against severe class imbalance?
  \item \textbf{RQ 3}: Can DNDS maintain it's performance with increasing database size?
  \item \textbf{RQ 4}: Which DNDS variant is the most effective overall?
\end{itemize}

\section{Related Work}
\label{sec:related work}
\subsection{Deep multimodal learning for municipal solid waste sorting}
Multimodal learning is a powerful techniquie for waste classification which significatly outperform unimodal approaches \cite{lu2022deep}.
Feature-level fusion of visual and acoustic data (using pre-trained VGG16 for images and 1D CNNs for audio) achieves 95\% accuracy compared to 85\% for unimodal approaches.
Critically, feature-based fusion in fully connected layers outperforms decision-level fusion when leveraging deeply learned representations. 
However reliability on limited dataset results in lack of sufficient sample diversity and waste variety, limiting model generalizability.
Systematic exploration of alternative modalities and fusion architectures is needed to push performance boundaries and understand which fusion strategies work best under different operating conditions.


\subsection{Retrieval Augmented Classification for Long-Tail Visual Recognition}
Large Transformer models store distributed visual representations across billions of parameters, creating a fundamental tension: comprehensive knowledge storage demands computational resources that exceed practical hardware limits.
Retrieval-augmented classification (RAC) resolves this constraint by decoupling model capacity from classification performance through explicit external memory \cite{long2022retrieval} .
This architectural separation enables dynamic knowledge updates - adding or removing classes without retraining model weights - a critical advantage for real-world deployment. 
RAC's effectiveness on balanced data distributions remains an open empirical question that warrants systematic investigation.

\subsection{KRA: K-Nearest Neighbor Retrieval Augmented Model for Text Classification}
KRA builds a memory of training sample embeddings, then during inference retrieves the top k nearest neighbors for a given input.
It augments these neighbors using both traditional techniques and LLM-based methods, weights them by similarity, and incorporates their labels into the final prediction.
The approach improves performance across models like BERT, RoBERTa, CNN, and LSTM—showing the largest gains for BERT and RoBERTa - and often surpasses baseline methods \cite{li2024kra}.
Searching neighbors across multiple dimensions or using multiple retrieval strategies could significantly reduce irrelevant matches.

\section{Dataset}

The CVPR 2024 trash classification dataset was used for training, validation, and testing of both the traditional and RAC approach. The task involves four-class classification of garbage categories: Black (3459 instances), Blue (6720 instances), Green (3492 instances), and TTR (3286 instances). Black refers to non-recyclable and non-compostable trash, Blue refers to recyclable trash, Green refers to compostable trash, and TTR refers to non-conventional trash like E-Waste. The dataset contains images of various trash, with the image file names being a short description of the image, acting as an additional modality.

The training subset has significant class imbalance, which presents a challenge for both the traditional approach and multimodal approach, as seen in \autoref{fig:train-set-distribution}.  

For experiments with RAC, the training subset of the dataset was converted into embeddings (vector representations) for both images and texts, and kept in a vector database, as examples. Chroma \cite{chromadb} was used for the vector database. During the experiments, the dataset was used in various ways, from training to simulating class imbalance, the details of which are in \autoref{sec:experiment setup}.

\subsection{Combining Modality}

In both the traditional approach and RAC approach, the two modalities - image and text were independently accounted for classification, with their individual classification weights combined for the final prediction, $\hat{y}$.

For traditional:
\begin{center}
  $W_{fused} = \alpha W_{text}+(1-\alpha)W_{image}$\\
  $\hat{y} = \softmax_{c}(W_{fused}$)
\end{center}

And, for RAC:
\begin{center}
  $S_{fused}(c) = \alpha S_{image}(c)+(1-\alpha)S_{text}(c)$\\
  $\hat{y} = \argmax_{c}(S_{fused}(c))$
\end{center}

While traditional works with class weights ($W_{image}, W_{text}$) and RAC works with class retrieval scores ($S_{image}(c), S_{text}(c)$), the underlying contribution hyperparameter $\alpha \in [0,1]$ has the same meaning. In traditional, softmax is used to convert weights to probabilities. In RAC, just the maximum score (argmax) is taken. The hyperparameter $\alpha$ started as 0.5, and was tuned when needed to extract the best classification performance.

\subsection{The Traditional Approach}

The traditional approach refers to the use of supervised deep learning methods for multimodal classification. For the image model, we selected MobileNetV3 as it offers a good balance between accuracy and efficiency \cite{howard2019searching}. For the classification head, a 3 layer neural network was used with $(960 \times 512)$, $(512 \times 256)$, and $(256 \times 4)$ layer sizes respectively. Since we used a pre-trained variant (on ImageNet dataset \cite{deng2009imagenet}) of MobileNetV3, we only needed to train the classification head (transfer learning). 

On the other hand, DistilBERT was used for classification. DistilBERT is a very lightweight transformer architecture model that shows good performance for text related tasks \cite{sanh2019distilbert}. Since our text labels for images are small, there would be no problem related to the small token window of DistilBERT. DistilBERT was paired with a linear classification layer with size of $(768 \times 4)$. Like MobileNetV3, DistilBERT is also a pre-trained model over Wikipedia \cite{wikidump} and BookCorpus \cite{Zhu_2015_ICCV} datasets, and only required the classification head to be fine-tuned.

\subsection{The RAC Approach}

The RAC approach works in the premise of vector similarity search, with cosine similarity being the key distance metric \cite{meng2025multimodal}. For a given query $q$, 10 nearest neighbors are retrieved separately for image and text modalities, and class scores are computed using different vector-based heuristics. Now, the scoring heuristics (for a given modality $m$ and a class $c$) used in this work are as follows:\\\\
Majority Vote:
\begin{center}
    $S_{m}(c) = S_m(c) = \sum_{i \in N_k^{(m)}(q)} \mathbf{1}[y_i = c]$
\end{center} 
Inverse Distance Weighting (IDW):
\begin{center}
      $S_{m}(c) = S_m(c) = \sum_{i \in N_k^{(m)}(q),y_{i}=c} \frac{1}{d_k^{m} + \epsilon} $
\end{center} 
Global Density Normalized Distance Score (Global DNDS):
\begin{center}
  $\rho_m(c) = \frac{N_m(c)}{\sum_{c'} N_m(c')}$ \\
  $S_m(c) = \sum_{\substack{i \in \mathcal{N}_k^{(m)}(q) \\ y_i = c}} 
  \frac{1}{\left(d_i^{(m)} + \epsilon\right)\,\rho_m(c)}$
\end{center} 
Local Density Normalized Distance Score (Local DNDS):
\begin{center}
  $\rho_m^{local}(c) = 
  \frac{\left| \{ i \in \mathcal{N}_{K_{density}}^{(m)}(q) : y_i = c \} \right|}
  {K_{density}}$,\\
  $S_m(c) = 
  \frac{
  \sum_{\substack{i \in \mathcal{N}_k^{(m)}(q) \\ y_i = c}} 
  \frac{1}{d_i^{(m)} + \epsilon}
  }
  {\rho_m^{local}(c)}$
\end{center}
Kernel Density Estimated Density Normalized Distance Score (KDE DNDS):
\begin{center}
  $\tilde{\rho}_m(c) = \sum_{j : y_j = c} 
  \exp\left( -\frac{\left(d_j^{(m)}\right)^2}{2h^2} \right)$, \\ \vspace{10pt}
  $S_m(c) =\frac{\sum_{\substack{i \in \mathcal{N}_k^{(m)}(q),,  y_i = c \frac{1}{d_i^{(m)} + \epsilon}}}} 
  {{\tilde{\rho}_m(c)}}$
\end{center} 

One can refer to \autoref{tab:notation} for the notations used throughout the paper. It is important to note here that all of the DNDS variants builds on the IDW strategy, where the cosine distance is inverted for a kind of ``similarity score''. DNDS variants normalize the score based on class density in different strategies.

Global DNDS is the most easiest strategy, where the normalization simply considers a class's overall density in the database. The higher the density for a class, the more imbalanced that class is, and the heavier the penalization for that class's score is. It is also the most efficient, with $O(1)$ complexity, as maintaining a counter for the classes can help in calculating $N_m(c)$ and $N_m(c')$ in constant time.

Local DNDS calculates the density over a local neighborhood, $ \{ i \in \mathcal{N}_{K_{density}}^{(m)}(q) : y_i = c \} $, near query $q$, instead of the entire database. This addresses a key weakness of Global DNDS - a class, $c$, may be globally sparse, but locally dense near $q$, which causes Global DNDS to over-amplify the weight of $c$, despite it's strong local presence. By fixating the density estimate to a local neighborhood, Local DNDS corrects itself for this locality problem. However, the neighborhood size $K_{density}$ is a very sensitive hyperparameter. If $K_{\text{density}}$ is too small compared to the database size and the spread of class instances, some classes will have near zero representations in the neighborhood, causing their density estimate to collapse. A more detailed discussion on fine-tuning the value of $K_{\text{density}}$ is discussed in \autoref{sec:experiment setup}.

KDE DNDS combines the strengths of Local DNDS and Global DNDS, by emphasizing more on local instances, while also keeping distant instances in consideration. It does not abruptly cut off neighborhood like a strict value of $K_{density}$ in Local DNDS, neither is it inconsiderate of local density like Global DNDS. KDE DNDS achieves this by implementing Kernel Density Estimation \cite{wkeglarczyk2018kernel}, with a Gaussian kernel. The Gaussian kernel allows nearby instances of class $c$ to have exponentially more contribution towards the density estimate, but also accounts for the further instances, even if exponentially less. This mirrors the Gaussian kernel in CNNs, where nearby pixels contribute more to defining a local patch — with the key distinction that KDE integrates over all database points rather than a fixed receptive field, which is exactly what eliminates the hard-cutoff artifact of Local DNDS. Using this as a normalization factor penalizes true majority classes more faithfully. The bandwidth $h$ controls the spatial scale at which class density is estimated — too small and the estimate becomes noisy; too large and KDE approximates the global prior, losing its local sensitivity. We address this through an ablation study in \autoref{sec:experiment setup}.


\section{Experiment Setup}
\label{sec:experiment setup}

All models were trained and evaluated under consistent settings. Hyperparameters such as fusion weight $(\alpha)$, neighborhood size $(k)$, and density parameters were tuned using validation set. Performance was measured using accuracy, macro F1-score, weighted F1-score, and per-class percision and recall.  

To evaluate robustness, controlled imbalance experiments were conducted by reducing class samples at predefined ratios. Additionally, a continual-memory setup was implemented where the retrieval database size was gradually increased to analyze performance trends. These experiments provide insights into the effectiveness of supervised and retrieval-based approaches in terms of accuracy, robustness, and generalization.

\section{Discussion}
Across standard evaluation settings, retrieval-augmented classification demonstrates a clear and consistent improvement over traditional supervised fusion models. While multimodal fusion effectively combines image and text information to produce string baseline results, it relies entirely on learned representations and may not fully capture complex relationships in the data. In contrast, retrieval-based approaches leverage embedding neighborhood, allowing predictions to be influenced by similar examples. Among the evaluated methods, density-aware heuristics such as global DNDS and KDE-DNDS achieve the highest performance, indicating that incorporate class density into scoring leads to more reliable and discriminative predictions. Simpler approaches such as majority voting and inverse distance weighting show comparatively lower performance, highlighting their limitations in handling overlapping class distributions.

Under class imbalance conditions, all methods experience a decline in performance; however, the extent of degradation varies significantly. Density-normalized approaches maintain substantially higher F1 scores for minority classes, particularly TTR, demonstrating improved robustness. This behavior suggests that density-aware methods effectively compensate for uneven class distributions by reducing the dominance of majority-class neighbors during retrieval. In contrast, non-normalized methods tend to bias predictions towards majority classes, leading to increased misclassifications. The confusion matrix analysis further supports these findings, showing that density-based methods achieve better class separation and reduced confusion between visually or semantically similar categories.

In the continual-memory experiments, performance improves as the size of the retrieval database increases, confirming the importance of richer neighborhood information. As more samples are added, the embedding space becomes more representative of the overall data distribution, enabling more accurate retrieval and classification. Density-based methods exhibit more stable and consistent improvements compared to other approaches,indicating their ability to effectively utilize additional data without being adversely affected by redundancy or imbalance. This highlights a key advantage or retrieval-based system: their performance can scale with data availability without requiring retaining.

The observed performance differences can be attributed to how each method handles local and global data structure. Density-aware heuristics explicitly normalize scores based on class distribution, reducing bias and improving generalization. In contrast, simpler heuristics rely solely on distance or count-based measures, which may not adequately reflected the true distribution of classes in the embedding space. Additionally, retrieval-based methods introduce flexibility by allowing the system to adapt dynamically to new data through memory expansion, unlike traditional models that require retraining for updates.

Overall, while supervised multimodal fusion provides a strong and efficient baseline, retrieval-augmented classification particularly with density normalization offers improved robustness, scalability, and generalization. These findings suggest that combining inference-based learning with retrieval-driven reasoning is a promising direction for multimodal classification tasks. Future work may explore more advanced embedding techniques, adapting retrieval strategies, and hybrid models that further integrate supervised learning with retrieval mechanisms to enhance performance accross diverse real-word scenarios.

\section{Conclusion}
\label{sec:conclusion}
In this work, we investigated the effectiveness of supervised multimodal fusion and retrieval-augmented classification for garbage classification. We found that while supervised fusion models provide strong baseline performance, retrieval-based approaches, particularly density-aware methods, achieve improved accuracy and robustness. These methods demonstrate better performance under class imbalance and benefit from increased retrieval memory, indicating their scalability and adaptability.

Overall, the results suggest the retrieval-augmented classification offers a more reliable and generalizable approach compared to traditional inference-based models. This highlights the importance of integrating retrieval mechanisms with multimodal learning for real-world classification tasks.
