\begin{abstract}

With the increasing use of multimodal data and specialized models for classification tasks, important questions arise regarding the effectiveness of traditional inference-based models versus retrieval-augmented approaches, especially when it comes to data skewness and domain drift. In this work, we compare supervised multimodal models with retrieval-augmented classification using 3 novel vector-based heuristics for a four-class garbage classification problem. Through our various experiments, we found that, while supervised fusion models provide strong baseline performance and inference velocity, retrieval-based methods, especially density-aware approaches, show improved robustness and better performance (upto 4\% in F1 Score, without any additional tuning).

\end{abstract}

\section{Introduction}
\label{sec:intro}

Garbage classification is an intriguing field that offers a good opportunity for applied deep learning (DL) multimodal classification tasks, as waste management facilities regularly use machine learning \cite{alexakis2025enhanced} to sort garbage according to some predefined criteria. By leveraging complementary information across modalities, multimodal approaches often outperform unimodal systems \cite{alexakis2025enhanced}. However, challenging conditions such as class imbalance, limited training data or domain drift often require rigorous training, normalization, complex feature engineering, augmentation, or re-training an existing model. This adds a significant computational cost \cite{islam2026learning}.

An alternative approach is retrieval-augmented classification (RAC), which utilizes embeddings to retrieve similar instance and make predictions based on neighborhood evidence. This allows to incorporate additional contextual information beyond training, but introduces challenges in retrieval strategies. A systematic comparison between traditional supervised inference and retrieval-based approaches remains limited, despite the potential. In our work, a two-phase experiment evaluates both strategies for multimodal garbage classification, with experiments conducted under standard settings, class imbalance, and varying instance size to analyze performance, robustness, and generalization. Implemented with modern vector databases, RAC only has a storage overhead, which is much cheaper than memory in terms of hardware price. 

To address the challenges in retrieval strategies, we propose a novel heuristic - Density Normalized Distance Score (DNDS) that specifically addresses the challenge of data skewness or imbalance, and domain drift by taking into account class distributions in various ways. This DNDS has 3 variants that we studied in this work - Global DNDS, Local DNDS, and KDE DNDS (discussed in \autoref{sec:methodology}). This DNDS heuristic is dynamic, designed to work with existing cosine heuristics \cite{xie2023brief}, and corrects itself for class skewness. Throughout this study, we evaluated these 3 variants of the DNDS heuristic, with a traditional deep learning involving a 2 step approach implemented with MobileNetV3 and DistilBERT. Now, our research questions (RQs) for this study are as follows:  

\begin{itemize}
  \item \textbf{RQ 1}: Does DNDS have better performance compared to traditional deep learning methods for classification? 
  \item \textbf{RQ 2}: How robust is DNDS against severe class imbalance?
  \item \textbf{RQ 3}: Can DNDS maintain it's performance with increasing database size?
  \item \textbf{RQ 4}: Which DNDS variant is the most effective overall?
\end{itemize}

\section{Related Work}
\label{sec:related work}
Deep multimodal learning techniques are highly efficient for waste classification, outperforming single-modality approaches. 
Feature-level fusion in fully connected layers outperforms decision-level strategies when leveraging deeply learned representations for waste sorting \cite{lu2022deep}, with late fusion demonstrating superior performance over early and intermediate fusion architectures \cite{bihler2023multi, willis2025exploring}. 
These findings establish logit-level fusion as the state-of-the-art baseline for multimodal waste classification.

Retrieval-augmented classification (RAC) leveraging k-NN retrieval during inference demonstrates consistent improvements over supervised baselines \cite{li2024kra}.
However, supervised approaches face critical limitations when deployed on imbalanced real-world waste data. 
RAC is particularly effective for minority classes in imbalanced datasets and domain adaptation scenarios \cite{long2022retrieval, lakatos2025investigating}, though shared representations for classification and retrieval hurt performance and lead to training instability \cite{liang2023retrieval}.
This motivates the exploration of multiple density-aware scoring heuristics to improve retrieval-based classification robustness.

To address class overlap in imbalanced neighborhoods, density normalization strategies like adaptive k-nearest neighbor methods have proven effective \cite{yuan2021novel,kim2021normalized}.
More recently, proximity ratio techniques combined with density-aware weighting schemes have shown improvements on imbalanced classification tasks \cite{amer2025effective}.

While ChromaDB has become standard for RAC pipelines, its application to classification tasks with density-aware scoring remains unexplored.
Additionally, no prior work has systematically evaluated multimodal RAC robustness under controlled imbalance stress tests or characterized performance scaling with memory growth.


\section{Dataset}

The multimodal garbage dataset by Cazarin Filho et. al. \cite{cazarin2025garbage} was used for training, validation, and testing of both the traditional and RAC approach. The dataset has 4 categories: Black (3459 instances, non-recyclable/non-compostable items), Blue (6720 instances, recyclable items), Green (3492 instances, compostable items), and TTR (3286 instances, other/e-waste items). The consists of images of various trash, with the image file names being a short description of the image, acting as an additional modality.

The training subset has significant class imbalance, which presents a challenge for both the traditional approach and multimodal approach, with nearly twice the instances of Blue, as seen in \autoref{fig:train-set-distribution}.  

Chroma \cite{chromadb} was used for the vector database. During the experiments, the dataset was used in various ways, from training to simulating class imbalance, the details of which are in \autoref{sec:experiment setup}.

\subsection{Combining Modality}
\label{subsec:combining modality}

In both the traditional approach and RAC approach, the two modalities - image and text were independently accounted for classification, with their individual classification weights combined for the final prediction, $\hat{y}$.

For traditional:
\begin{center}
  $W_{fused} = \alpha W_{text}+(1-\alpha)W_{image}$\\
  $\hat{y} = \softmax_{c}(W_{fused}$)
\end{center}

And, for RAC:
\begin{center}
  $S_{fused}(c) = \alpha S_{image}(c)+(1-\alpha)S_{text}(c)$\\
  $\hat{y} = \argmax_{c}(S_{fused}(c))$
\end{center}

While traditional works with class weights ($W_{image}, W_{text}$) and RAC works with class retrieval scores ($S_{image}(c), S_{text}(c)$), the underlying contribution hyperparameter $\alpha \in [0,1]$ has the same meaning.

\subsection{The Traditional Approach}

The traditional approach refers to the use of supervised deep learning methods for multimodal classification. For the image model, we selected MobileNetV3 as it offers a good balance between accuracy and efficiency \cite{howard2019searching}. This model is based on the CNN (Convolutional Neural Network) architecture.

On the other hand, DistilBERT was used for classification. DistilBERT is a very lightweight transformer architecture model that shows good performance for text related tasks \cite{sanh2019distilbert}. Since our text labels for images are small, there would be no problem related to the small token window of DistilBERT.

\subsection{The RAC Approach}

The RAC approach works in the premise of vector similarity search, with cosine distance being the key distance metric \cite{meng2025multimodal}. For a given query $q$, 10 nearest neighbors are retrieved separately for image and text modalities, and class scores are computed using different vector-based heuristics. Now, the scoring heuristics (for a given modality $m$ and a class $c$) used in this work are as follows: \\\\
Majority Vote:
\begin{center}
    $S_{m}(c) = S_m(c) = \sum_{i \in N_k^{(m)}(q)} \mathbf{1}[y_i = c]$
\end{center} 
Inverse Distance Weighting (IDW):
\begin{center}
      $S_{m}(c) = S_m(c) = \sum_{i \in N_k^{(m)}(q),y_{i}=c} \frac{1}{d_k^{m} + \epsilon} $
\end{center} 
Global Density Normalized Distance Score (Global DNDS):
\begin{center}
  $\rho_m(c) = \frac{N_m(c)}{\sum_{c'} N_m(c')}$ \\
  $S_m(c) = \sum_{\substack{i \in \mathcal{N}_k^{(m)}(q) \\ y_i = c}} 
  \frac{1}{\left(d_i^{(m)} + \epsilon\right)\,\rho_m(c)}$
\end{center} 
Local Density Normalized Distance Score (Local DNDS):
\begin{center}
  $\rho_m^{local}(c) = 
  \frac{\left| \{ i \in \mathcal{N}_{K_{density}}^{(m)}(q) : y_i = c \} \right|}
  {K_{density}}$,\\
  $S_m(c) = 
  \frac{
  \sum_{\substack{i \in \mathcal{N}_k^{(m)}(q) \\ y_i = c}} 
  \frac{1}{d_i^{(m)} + \epsilon}
  }
  {\rho_m^{local}(c)}$
\end{center}
Kernel Density Estimated Density Normalized Distance Score (KDE DNDS):
\begin{center}
  $\tilde{\rho}_m(c) = \sum_{j : y_j = c} 
  \exp\left( -\frac{\left(d_j^{(m)}\right)^2}{2h^2} \right)$, \\ \vspace{10pt}
  $S_m(c) =\frac{\sum_{\substack{i \in \mathcal{N}_k^{(m)}(q),,  y_i = c \frac{1}{d_i^{(m)} + \epsilon}}}} 
  {{\tilde{\rho}_m(c)}}$
\end{center} 

Note that, all of the DNDS variants builds on the IDW strategy, where the cosine distance is inverted for a ``similarity score''.

With Global DNDS, normalization simply considers a class's overall density in the database. The higher the density for a class, the more imbalanced that class is, and the heavier the penalization for that class's score is. It is also the most efficient, with $O(1)$ complexity, as maintaining a counter for the classes can help in calculating $N_m(c)$ and $N_m(c')$ in constant time.

Local DNDS only calculates the density over a local neighborhood, $ \{ i \in \mathcal{N}_{K_{density}}^{(m)}(q) : y_i = c \} $, near query $q$. This addresses a key weakness of Global DNDS - a class, $c$, may be globally sparse, but locally dense near $q$, which causes Global DNDS to over-amplify the weight of $c$, despite it's strong local presence. Local DNDS corrects itself by fixating the density estimate in a locality. However, the neighborhood size $K_{density}$ is a very sensitive hyperparameter. If $K_{density}$ is too small compared to the database size and the spread of class instances, some classes will have near zero representations in the neighborhood, causing their density estimate to collapse (discussed more in \autoref{sec:experiment setup} and \autoref{sec:results}).

KDE DNDS combines the strengths of Local DNDS and Global DNDS. It does not abruptly cut off neighborhood like a strict value of $K_{density}$ in Local DNDS, neither is it inconsiderate of local density like Global DNDS. KDE DNDS achieves this by implementing Kernel Density Estimation \cite{wkeglarczyk2018kernel}, with a Gaussian kernel. The Gaussian kernel allows nearby instances of class $c$ to have exponentially more contribution towards the density estimate, but also accounts for the further instances. This mirrors the Gaussian kernel in CNNs, where nearby pixels contribute more to defining a local patch - although KDE integrates over all database points rather than a fixed receptive field, which is exactly what eliminates the fixed $K_{density}$ of Local DNDS. The bandwidth $h \in (0, 1]$ controls the spatial scale at which class density is estimated - too small and the estimate becomes noisy; too large and KDE loses its local sensitivity. We address this through an ablation study in \autoref{sec:experiment setup}.


\section{Experiment Setup}
\label{sec:experiment setup}

The experiment was done in 2 phases. Phase 1 involved the traditional methods, and phase 2 involved the RAC heuristics. Various metrics were used to evaluate performance, with emphasis being on accuracy, F1 score, and inference time (in milliseconds). Since F1 score cumulates precision and recall, it is an ideal metric for measuring classification robustness, especially with training data imbalance. Hence, some results are reported purely based on F1 score. All of the reported results in \autoref{sec:results}, was ran on the test set. After both phases, the results were analyzed to answer all the RQs.

\subsection{Phase 1}

Here, MobileNetV3 was paired with a 3 layer FCNN (Fully Connected Neural Network) classification head with layer sizes of $(960 \times 512)$, $(512 \times 256)$, and $(256 \times 4)$ respectively. During training, various image transformation strategies were implemented for robustness, such as - flipping, rotation, color jitter, and affine. Since we used a pre-trained variant (on ImageNet dataset \cite{deng2009imagenet}) of MobileNetV3, we only needed to train the classification head (transfer learning \cite{si2023apple}).

DistilBERT is also a pre-trained model over Wikipedia \cite{wikidump} and BookCorpus \cite{Zhu_2015_ICCV} datasets, and only required the classification head to be fine-tuned. For this experiment, a linear FCNN classification head was paired with DistilBERT - which is a very common set-up, when using the sequence classifier variant of the model, available through HuggingFace Transformers library in python \cite{huggingface_distilbert}. No further text transformation techniques were used here, as such augmentations require special techniques \cite{bayer2022survey}, and it is beyond the scope of this study. 

After individual training of the text and image models, the modality weight, $\alpha$ was fine-tuned with the validation set and then a final prediction with the best $\alpha$ was made on the test set according to \autoref{subsec:combining modality}. 

\subsection{Phase 2}

Phase 2 was done in 5 steps. In the first step, the test data was ingested into the vector database, with image and text having different collections. With each entry, it's class label was included as ``label'' metadata. OpenClip VIT-B-32 was used for the image embedding function as this variant has low resource requirements and high inference speed \cite{cazarin2025multimodal, chen2025standalone} and is default for ChromaDB. For text embeddings, All-MiniLM-L6-v2 was used for the similar reasons of low resource and speed. 

In both the second and third step, the RAC approach was evaluated with the test data in a similar manner. However, for step 3, $\alpha$ was not fine-tuned, and stayed at it's default of 0.5. This provided a good baseline for an out-of-box usage of the RAC approach.

In the fourth step, the RAC approaches were tested with severe imbalance of class instances, from 2:1 to 10:1 for majority:minority. To simulate imbalance in a real-world scenario, a separate vector database collection was constructed with the train set for every ratio, by under-sampling the minority classes according to the ratio. This addresses RQ 2.

Finally, in the fifth step, the RAC approaches were tested for continual learning, to elucidate RQ 3. In this experiment, a vector database collection was iteratively populated with the test data, from 10\% of the total data to 100\% of the total data - which mimics a growing database of instances.

\section{Conclusion}
\label{sec:conclusion}
In this work, we investigated the effectiveness of supervised multimodal fusion and retrieval-augmented classification for garbage classification. We found that while supervised fusion models provide strong baseline performance, retrieval-based approaches, particularly density-aware methods, achieve improved accuracy and robustness. These methods demonstrate better performance under class imbalance and benefit from increased retrieval memory, indicating their scalability and adaptability.

Overall, the results suggest the retrieval-augmented classification offers a more reliable and generalizable approach compared to traditional inference-based models. This highlights the importance of integrating retrieval mechanisms with multimodal learning for real-world classification tasks.
